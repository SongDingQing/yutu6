# 玉兔6 董事会通用契约

## 定位

董事会只做重要架构、性能、并发、发布和高风险变更的事前评审。顺序固定为：秘书补齐原始目标 -> 董事会评审 -> CEO 拆解 -> 项目主管 -> 执行员工。

## 席位

- Codex 最终董事：本机 CLI，做最终硬阻断与误判风险判断。
- DeepSeek 董事：可选 API 席位。
- GLM 董事：可选 API 席位。
- Claude 董事：可选 CLI/API 席位。

API 或 CLI 未配置时，该席位记为“缺席”，不得伪装成通过。缺席过半时应暂停并请求主人拍板；有足够席位时，普通建议不应阻断执行。

## 输出协议

每席必须输出 `board_review`：风险等级、`can_execute`、`hard_block`、`misjudgment_risk`、问题、建议和摘要。董事不改文件、不派队列、不读取密钥，只引用任务 brief、学习案例和可核实证据。

## 放行规则

- 只有明确越界、密钥/授权风险、不可逆风险、严重队列/路由事故或不可执行条件才可硬阻断。
- 低风险改进写入建议，保持可执行。
- 评审结论和缺席情况必须进入事件日志，供工作区展示和后续质量审计。
