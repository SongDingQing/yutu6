# 玉兔6 该加什么功能 · 轻重缓急(秘书建议)

> 老板要求:加切实提高生产力的功能、不要花里胡哨、专注实用,目标是「多智能体 + 可信赖的生产力平台」。这是我的排序建议。2026-06-20。

## 一句话核心判断
**让它成为"可信赖的生产力平台",杠杆不在"更多智能体/更炫的界面",而在三件事**:
1. **让"完成"可信** —— 验收闭环 + 审计追踪(你能信它说的"done");
2. **从"只会写代码"扩到"做你的真实工作"** —— 连接器 + 技能(邮件/文档/数据/research,不只码 UI);
3. **你能放手不盯着** —— 异步自治 + 审批收件箱 + 成本护栏。

办公室视觉/像素美化:好看,但**不提生产力**,属锦上添花,建议地基稳之前先收着。

---

## P0 · 可信赖地基(没这些谈不上"信赖",最先做)
1. **验收闭环(Acceptance Gate)** —— 每个任务带"完成标准",产出必须通过校验(自动测试/截图对照/复核)才算 done,不过就回炉。**这是"可信赖"的心脏**:你能信"完成"两个字。(现在"真实完成状态"刚修好,下一步是"完成前先验收"。)
2. **真实审计追踪** —— 每个任务自动生成一句人话:"这次改了什么/没做什么/验证结果",带 diff,可回溯可回滚。→ 你能复核、能信任、出事能查。
3. **自动备份 + 一键恢复(常驻)** —— 定时快照 + 异地冷备(脚本已有,缺常驻托管)。→ 不丢活。
4. **成本护栏** —— 预算上限、额度告警(快烧完提前提醒)、消耗可观测(claude/codex 订阅也要可见)。→ 不偷偷烧钱。
5. **测试网自动门 + 健康告警** —— 改核心必过测试(已有 7 个测试,缺自动触发);队列堆积/失败率/卡死主动飞书告警。→ 改不坏、出事第一时间知道。

## P1 · 核心生产力(让它真能替你干活,这是价值大头)
6. **从"写代码"扩到"你的真实工作"** ⭐ —— 接**连接器**(邮件/日历/文档/飞书/表格/网盘)+ 配**技能**,让它能:写研究报告、起草文档/邮件、整理数据/表格、收件箱三筛、会议纪要。**现在它主要只会改 UI 代码——这是生产力最大的缺口**。一个只会码自己界面的系统,提不了你的生产力;能处理你日常知识工作的,才是平台。
7. **记忆真正注入(RAG + memory 打通)** —— 干活前自动调你的偏好/项目/历史(现在记忆建了但 agent 没接检索)。→ 你不用反复交代背景,它越用越懂你。
8. **审批收件箱(human-in-the-loop)** —— 需要你拍板的(发送/发布/花钱/不可逆)集中成**一个收件箱、批量快速批**,而不是逐条打断你。→ 你掌控关键动作,但不被频繁打扰。
9. **异步自治 + 主动复现工作** —— 定时简报/监控/digest(定时任务底子已有)、夜间自主推进 + 飞书汇报(已在做)。→ 你睡觉它干活、它替你盯着。

## P2 · 吞吐与规模(地基稳了再上)
10. **有限并行** —— 按"资源域"切分 + 单写者:域内串行、域间并行(UI/知识库/素材/研究 各跑各的),提吞吐。
11. **路由/拆解可靠性彻底化** —— 确定性优先路由 + node 失败自动重试(在修),让任务成功率稳。
12. **多项目深化** —— 控制台 + Simulaid + 后续项目,项目隔离 + 复用。

## P3 · 锦上添花(明确次要,别占地基的资源)
13. **手机「元宵」端** —— 远程指挥/看进度(借 cc-connect/OpenAgents 思路)。实用,但排在地基+核心生产力之后。
14. **视觉/办公室美化** —— 好看,但**不提生产力**。建议:地基(P0)稳之前,把生图/像素这类先**收着**,省时间省额度。

---

## 我的建议:先做这 3 件(性价比最高)
1. **验收闭环 + 审计追踪**(P0-1/2)—— 一次性把"可信赖"立住:done 是真的、改了啥看得见。
2. **接 1~2 个连接器 + 对应技能**(P1-6)—— 让它从"码 UI"跨到"做你的真实工作"(建议先接**邮件或文档**,最高频)。
3. **成本护栏 + 自动备份常驻**(P0-3/4)—— 不丢活、不烧钱,放手的前提。

做完这 3 件,玉兔6 就从"一个会自我改造的炫酷系统"变成"**你敢放手、能替你干真活的生产力平台**"。其余按 P1→P2→P3 推进。

> 一句话总结:**先让它可信(验收+审计+备份+成本),再让它有用(连真实工作+记忆注入+审批收件箱),最后才谈快(并行)和远(手机端)。视觉先放放。**
