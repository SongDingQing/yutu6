## 五、失败教训(详见 `memory/experience.md`,核心几条)
- **假完成遍地**:复审"软"(看声明不核实)、没硬测试 → 标 done 没真做,完成率曾 <20%。解药:done gate + 逻辑链 + 结构化验收表 + 截图自验(opus 识图)。
- **codex 烧太快**:董事会失控(10832 事件)+ 维修单多 + 失败重做 + harness 执行层。**省 codex = 系统硬化是同一件事**。
- **重启即清扫,清扫即杀活**:watchdog 误判重启 → 批量 SIGTERM 杀活引擎 → 失败刷屏。控制台重启失败常因 http-probe 2500ms 超时误判。
- **记忆被跨过**:设计都在 decisions.md,但执行 agent 不读不对照。解药:结构化验收表(要点取自设计,逐项核)。
- **补丁落盘 ≠ 运行态生效**:改了代码要 worker 重启/新 engine 才生效。
- **GLM 是纯文本 runner、不能落盘**:要落盘的任务别派 GLM,用 codex/harness。
