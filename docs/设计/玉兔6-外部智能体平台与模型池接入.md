# 玉兔6外部智能体平台与模型池接入

> 2026-07-21 更新：本文件保留为最初 sidecar 方案的历史记录。新的统一前门、
> 替换路径与运行契约以 `玉兔6-模型与智能体中枢-架构与迁移.md` 为准。

## 目标

在不替换玉兔6文件队列、角色路由和 done gate 的前提下，让 Nexent 等外部
Agent 平台可以被发现、按需启动并作为执行节点接入；所有模型继续优先通过
玉兔6现有 new-api 模型池统一管理和计量。

## 决策

| 项目 | 决策 |
|---|---|
| 主控制面 | 玉兔6原生文件队列 |
| 外部平台协议 | A2A Agent Card + REST/JSON-RPC |
| 外部平台启动 | 按需、显式确认、禁止自动开机 |
| 模型凭据 | 继续保存在玉兔6本地配置；外部平台只拿内部令牌 |
| 完成判定 | 外部输出是证据，不是可信 done verdict |
| Nexent Web 端口 | `3100`，避开 new-api 的 `3000` |
| Nexent A2A 端口 | `5013` |

## 为什么不直接替换

Nexent 的优势是零代码 Agent 创建、A2A、MCP、技能、记忆、知识库、版本和
市场；代价是一整套微服务和数据基础设施。玉兔6已经有文件队列、角色组织、
本地模型池、证据链和完成门禁。两者应通过协议协作，而不是把成熟的本机控制面
推翻重做。

## 实施阶段

### 已完成

1. 建立平台单一目录和资源/端口/信任策略。
2. 建立零依赖平台管理命令：列表、模型清单、doctor、plan、prepare、start、stop。
3. 建立零依赖 A2A 客户端，支持 Agent Card、REST 和 JSON-RPC。
4. 将 Nexent 设为可选 sidecar，并处理 `3000 -> 3100` 端口冲突。

### 启用 Nexent 后

1. 在 Nexent 中将 OpenAI-compatible provider 指向
   `http://host.docker.internal:3000/v1`。
2. 用本地内部令牌完成模型测试，不复制厂商裸 key。
3. 发布一个只读测试 Agent 为 A2A。
4. 用 `a2a-client.js discover/run` 做闭环。
5. 只有在真实任务证据经过玉兔6 review/done gate 后，才登记为完成。

## 回滚

外部平台适配层与玉兔6核心无共享运行状态。停止或删除 Nexent 不影响原生队列；
删除 A2A runner 配置即可恢复到纯玉兔6执行。
