# 玉兔6统一启动与重启恢复

## 目标

设备重启并登录 `yutu6` 用户后，由一个入口恢复玉兔6全部生产组件。各组件仍由
自己的 LaunchAgent 或容器重启策略负责保活，统一启动器只做幂等安装、拉起和健康核对，
不另起重复的 server、worker 或 engine。

## 单一入口

```bash
cd /Users/yutu6/玉兔6工作区
bash start-all.sh start
bash start-all.sh status
```

机器可读清单在
`projects/控制台/config/startup-components.json`。执行状态在
`projects/控制台/artifacts/startup/status.json`，日志在同目录。

## 自动启动

```bash
bash projects/控制台/tools/install-unified-startup-launchd.sh
```

安装 `com.yutu6.startup` 用户 LaunchAgent：

- `RunAtLoad=true`：用户登录后立即恢复。
- 每 600 秒轻量核对一次：处理 OrbStack 启动竞态或偶发组件退出。
- 单飞锁防止两次核对重叠。
- macOS GUI、TCC、Keychain 和用户级密钥均要求用户会话，因此采用 LaunchAgent，
  不采用 root LaunchDaemon。

## 纳管组件

核心组件：

- OrbStack 与 `new-api`、`zhipu-coding-proxy` 容器。
- 玉兔6控制台 `com.yutu6.console`。
- Hermes/飞书网关 `ai.hermes.gateway`。
- Hermes 本地语音监听 `com.yutu.hermes.voicewake`。
- 元宵决策拉取、任务快照推送。

低成本维护：

- 控制台 watchdog。
- RAM watchdog。
- 长期运行维护。
- 孤儿 worker 清理。
- 每日治理定时入口。

明确不纳管：

- `frontend-canary`：有固定时长的发布观察任务。
- `repair-memory-maintenance`：旧入口，已被 RAM/长期维护替代。
- 独立 queue worker：由控制台 server 监督，单独启动会产生重复 worker。

## 回滚

只卸载统一协调器，不停止现有组件：

```bash
bash projects/控制台/tools/install-unified-startup-launchd.sh --unload
```

原有各 LaunchAgent 和容器仍可独立运行。若需修改组件清单，先改
`startup-components.json`，执行 `bash start-all.sh validate`，再重新安装。
