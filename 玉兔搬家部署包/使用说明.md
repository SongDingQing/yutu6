# 使用说明 · 新 Mac mini 一键部署

这个文件夹是个「即插即用」包。把它整份拷到新 Mac mini,运行里面的脚本,就会自动装好
**Homebrew、Node、Git、Python、Claude Code、Codex**,并在主目录建立**「玉兔6工作区」**(多智能体目录)。

---

## 三步走

### 第 1 步:拷过去
把**整个**「玉兔6-新机部署包」文件夹拷到 U盘 → 插到新 Mac mini → 拷到桌面。
⚠️ 要拷**整个文件夹**,不要只拷 `安装.command` 一个文件(脚本需要旁边的模板)。

> (可选)想把现在的 **v2rayN 代理**一起带过去:拷之前先看 `代理/说明.txt`,把 v2rayN 和你的节点配置放进 `代理/` 文件夹,安装时脚本会问你是否启用。

### 第 2 步:运行安装脚本
任选一种,**方法一最稳**:

- **方法一(推荐,最稳):**
  打开「终端」(按 `Command+空格`,输入 `Terminal` 回车)。
  在终端里先输入 `bash` 和**一个空格**,然后把 `安装.command` **拖进**终端窗口,回车。

- **方法二(图省事):**
  直接**双击** `安装.command`。
  若提示「无法打开,因为来自身份不明的开发者」,改用方法一即可。

> 运行中可能要求**输入开机密码**(装 Homebrew 时)。
> 输密码时屏幕**不显示任何字符**,这是正常的,输完直接回车。
> 全程需要联网,大约 10–20 分钟(取决于网速)。

### 第 3 步:登录两个工具(各一次)
脚本跑完后,按屏幕提示,在终端里:

1. 输入 `claude` 回车 → 登录 **Anthropic** 账号(需付费套餐)。
2. 输入 `codex` 回车 → 用 **ChatGPT 账号或 API Key** 登录。

> 若提示 `command not found`,关掉终端窗口、**重开一个**再试(PATH 需要重新加载)。

---

## (可选)带上你的 v2rayN 代理

如果在 `代理/` 里放了 v2rayN(怎么放见 `代理/说明.txt`),安装到一半会问你:

> 是否启用代理配置(安装 v2rayN + 导入你的节点)?[y/N]

选 `y`:把 v2rayN 装到 `~/Applications`、解除 macOS 隔离、并把你的节点(VLESS / TUIC)还原到位,打开就能用。
不放或选 `N`:整步跳过,不影响其他安装。

⚠️ 节点配置含私钥 / 密码,U盘自己保管好。

## 这个包里有什么

| 文件 / 文件夹 | 作用 |
|---|---|
| `安装.command` | 一键安装脚本(双击或拖进终端运行) |
| `使用说明.md` | 就是本文件 |
| `docs/多智能体架构设计.md` | 系统设计文档(四层组织 + 上下文管理) |
| `workspace-模板/` | 「玉兔6工作区」的目录骨架和模板,安装时复制到主目录 |
| `代理/` | (可选)放入 v2rayN 和你的节点配置,安装时可选择启用 |

装完后,工作区在:**`~/玉兔6工作区`**(也就是「用户目录/玉兔6工作区」),
已用 git 初始化,改动可回滚。先读里面的 `README.md` 和 `docs` 设计文档。

---

## 可重复运行
脚本是**幂等**的——已经装好的东西会自动跳过,可以放心多跑几次。
已存在的 `~/玉兔6工作区` **不会被覆盖**。

## 出问题怎么办
把终端里**红色 ✗** 那几行**截图**发我,我来定位。
脚本是纯文本,可以用「文本编辑」打开逐行查看,确认安全后再运行。

## 安全提示
只从你信任的来源(我们这次对话)运行这个脚本。它只做:装上述软件、建工作区目录,不碰你的其他文件。
