# 知识库 × 媒体生成 联动设计(任务:知识库打通 + 联动生图/PPT/视频)

> 2026-06-25 · 维修主管出稿(4 路并行调研 workflow 综合)· 老板原话:"做一个知识库打通,我希望知识库和生图生PPT生视频联系在一起"

## 核心
知识库 = 媒体生成的**单一真相源 + 检索增强前置层**。存四类高价值载荷:提示词模板、品牌/风格准则(yuhua #C00000+雅黑、办公室生图规范)、历史优秀案例(few-shot)、工作准则/门禁。
数据流:任务信封 →(阶段1)kb 检索 top-k 注入"## 知识库检索"只读块 → 生图/PPT/视频前把命中模板+约束拼进生成参数 → 产物落 artifacts → meta.json 回灌 kb 闭环。
- **知识库走"预取注入"**(纯文本 runner 没 shell 调不动 query.py,预取对所有 runner 生效)。
- **媒体生成走"暴露子命令"**(动作型,有手 runner 主动调,挂 secretary-tools 工具总线)。两路径不混。

## 关键现状
- embedding 全缺(sqlite-vec 未装、Ollama 不可达)→ 只有 **FTS5 全文检索在跑**,但**够打通,第一步零依赖**。
- kb 只有 7 篇玩具种子,且 **ingest.py 读 wiki/ 不读 corpus/** → 第一步配套必须做一次真迁移。
- meowa 客户端齐全(生图/animate-run 动画),但生成命令没被 node 自动调;办公室素材受门禁冻结(reference-v3 待老板确认)。
- 生 PPT 无生成引擎(yuhua 只守门),需装 python-pptx。通用文生视频=零实现(Seedance 纸面)。

## 分阶段(标注依赖)
- **阶段0【现在·零依赖】** query.py 加 `--json`(输出层);整理素材进 knowledge/wiki/ 跑 ingest 重建 kb.sqlite。验收:`query.py "..." --json` 返合法 JSON、无结果时 ok:true。
- **阶段1【现在·碰引擎一处】** cli-runner buildEnvelope + fetchKbContext(spawnSync 超时+静默降级)注入检索块;secretary-tools 加 kb-query 子命令。验收:task.md 出现"## 知识库检索"块;query 不可用时正常生成不报错。
- **阶段2【现在·生成受门禁】** secretary-tools 加 gen-image(spawnMeowa 扩展)+ kbQuery 预取注入 requirement;过 office-image-template-check 门禁;真生成待 reference-v3 解冻。
- **阶段3【现在】** gen-animation(animate-run)+ kb 注入;hermes 扩视频回传白名单(webp/gif)。
- **阶段4【需老板拍板装 python-pptx】** shared/tools/deck/gen_pptx.py + gen-ppt 子命令 + yuhua 字体门 + peekaboo 验收。
- **阶段5【需老板补依赖+密钥】** Seedance 图生视频(纸面)。
- **阶段6【可选提质】** 装 sqlite-vec + embedding 端点 → FTS5 升向量语义召回(不改上层接口)。

## 风险
- 碰核心引擎仅阶段1 一处(buildEnvelope),spawnSync 超时+失败静默降级,绝不阻断信封组装。
- 真迁移未做则注入为空=联动没接(阶段0 必做)。
- 生图沿用 done gate(peekaboo 截图+opus 对照)防"失败报完成"历史。
