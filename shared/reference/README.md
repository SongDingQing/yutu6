## 监管误区库 = 怎么闭环

```
反复失败 / 重大漏洞
   │  (§10 attempt 计数命中阈值,默认 recurrence_N=3)
   ▼
按"问题指纹"聚类(产物 × 症状类别 × 反馈类型)  ── 监管SOP §2
   │
   ├─ 命中「失败案例库 / SOP §5 误区库」已有条目 → 直接按既有对策,不重复踩
   │
   └─ 未命中 → 走复盘七步(SOP §3)→ 产出「现象→根因→避免法」
            → 回填《失败案例库》+ 更新《SOP §5 误区库》+ 归档进知识库(§15)
            → 防复发规则交质量运营师硬化(§16),验收门挂 §10
```

## 两层的分工

- **误区(meta,在 SOP §5)**:跨案例的**认知误区名**——追症状不追根因 / 盲调不验证 / 只信自算忽略证据 / 几何忽略外溢 / 过早收敛。监管人员用它给问题**命名**。
- **案例(具体,在 失败案例库)**:某产物某症状的**具体踩坑 + 避免法**。先查这里,命中即用。

> 一句话:**案例库防"同一个坑再踩",误区库防"同一类错再犯"。** 效率问题归质量运营师(硬化),反复出错归监管人员(复盘)。
