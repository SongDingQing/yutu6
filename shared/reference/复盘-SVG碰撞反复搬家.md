# 复盘 · SVG 元素碰撞反复搬家(奠基案例)

> 监管误区库的**首个真实案例**(蓝图 §17)。监管人员制度本身就源于这次踩坑。
> 用途:① 演示《监管人员复盘SOP》模板怎么填;② 给 §5 误区库命名提供来源。

## 触发
- attempt_count = 多次(连线→发光→光点→贴字,每版换法)/ recurrence_count ≥ 3 / 重大漏洞:否
- 问题指纹:`某.svg` × `collision(碰撞/重叠/糊在一起)`

## 现场
- 产物 / 范围:一张 SVG,多个元素视觉重叠。
- 症状(用户原话 + 截图):元素"挤在一起 / 贴着 / 糊了";用户**给了错误截图**。

## 历次尝试(逐条)
| # | 做了什么 | 依据 | 结果 | 用户反馈 |
|---|---|---|---|---|
| 1 | 改最显眼那处连线坐标 | 截图里最扎眼 | 该处分开,别处又撞 | 还是糊 |
| 2 | 调发光元素位置 | 按几何边缘算间距 | glow 仍外溢重叠 | 没解决 |
| 3 | 移光点 / 贴字 | 继续按坐标推断 | 碰撞只是"换位置搬家" | 反复 |

## 命中的误区(选自误区库)
- **追症状不追根因**:每次只改截图里最显眼一处,没整区域两两排查。
- **几何忽略外溢**:只按几何边算间距,忽略 blur/glow 外溢量。
- **盲调不验证 / 只信自算忽略证据**:不渲染自查、只靠坐标推断,**没认真对照用户给的错误截图**。

## 根因
- 现象 → 根因 → 为何久治不愈:碰撞反复**不是某处坐标错,而是方法错**——追症状、按纯几何算、不渲染不看证据。改 N 次症状当然治不好,因为根因在认知方法,不在坐标。

## 对策
- 这次一次性修法:对重叠区域**所有元素对两两排查**间距;间距 = 几何边 + glow 外溢量 + 富余;**渲染出来**逐一核对。
- 防复发规则(交质量运营师硬化为 skill/检查项):**"改带视觉的产物,必须渲染自查 + 对照用户提供的截图,才算 done。"**
- 新增验收门(§10):视觉类产物 done 必须附**渲染截图**,且与用户证据对照通过。
