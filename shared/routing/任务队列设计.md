# 玉兔6 · 智能体任务队列设计

> 单独的队列层,独立于具体 agent。**老板叠加任务不必等**;支持**引导/插队/调换顺序**。
> 依据蓝图 §8 控制平面(命令队列 + steering/follow-up)。先给 CEO 用,再下沉为所有 agent 通用原语。
> 立项:2026-06-18。实现触碰 server.js/engine(Codex 正在开发)→ 由 CEO/主管派给开发落地;本文是设计。

## 0. 一句话
每个 agent 一条队列;老板把任务**叠加**进去就走,worker 串行消化;运行中可**引导**转向,排队中可**插队/调换顺序**。

## 1. 两种语义(§8)
- **follow-up(排队式,默认)**:加到队尾,当前任务做完才取下一个。**老板可连续叠加,无需等他跑完**。
- **steering(打断式 = 引导)**:对**正在跑**的任务注入一条引导消息,在**安全检查点**(工具执行后 / 下次模型调用前)送达,实现中途转向,不粗暴打断。

## 2. 队列项(entry)
```
{ id, target,            // 目标 agent/角色(ceo / 主管:控制台 / 员工 …)
  task,                  // 任务信封或目标文本
  priority,              // 0 普通 / 高(数字越大越先取)
  status,                // queued | running | paused | done | failed | canceled
  seq, enqueued_at,      // 单调序 + 时间(排序/审计)
  idem,                  // 幂等键(同任务不重复入队)
  steer: [] }            // 运行中注入的引导消息
```

## 3. 操作(老板/秘书可用)
| 操作 | 作用 |
|---|---|
| **enqueue 叠加** | 加到队尾,**立即返回**,不等待 |
| **引导 steer** | 给"正在跑的任务"加引导消息,安全点生效(转向/补充) |
| **插队 jump** | 把某 queued 项提到队首(或设高 priority,worker 先取) |
| **调换顺序 reorder** | 指定/拖动队列顺序 |
| **pause / resume / cancel** | 暂停 / 恢复 / 取消某项 |

## 4. worker(消费,每 agent 一个)
- 串行取队列(默认 **one-at-a-time**,§8);**高 priority 优先**,同级按 seq(FIFO)。
- 取一个 → 经引擎执行(信封运行时)→ 运行中在检查点消费 `steer` → done 取下一个。
- 单写:一个 agent 同时只跑一个任务(避免互相覆盖)。

## 5. 存储与接口
- **文件系统队列**(原子、可审计,沿用自优化 dev-worker 那套):`projects/控制台/artifacts/queues/<agent>/<priority>-<seq>-<id>.task`。
  - 插队/reorder = 改文件名 priority/seq 前缀;steer = 往 `…/running/<id>.steer/` 投消息;状态迁移 = mv 到 `proc/ done/ failed/`。
- **API(server.js 暴露)**:`POST /api/queue/<agent>`(enqueue)、`GET /api/queue/<agent>`(列表)、`POST /api/queue/<agent>/<id>/{jump,reorder,steer,cancel,pause}`。
- 每步写引擎事件日志(`engine-events.jsonl`)→ WebUI 实时可观测。

## 6. WebUI(调度页 + 队列面板)
- 每个 agent 一栏"待办队列":显示 running / queued / done;可**拖动 reorder**、**插队**按钮、**引导**输入框(注入 steer)、**取消**。
- 老板在派单框连发多条 → 自动叠加进 CEO 队列,不阻塞。

## 7. 落地次序
1. **CEO 先有队列**:老板叠加任务不等(最高频需求)。
2. **下沉为通用原语**:主管→员工、质量运营、监管、自优化都复用同一套队列(自优化的 `dev-worker.sh` 已是本队列的一个文件系统实例,可作参照/统一)。
3. WebUI 队列面板(可与"工作区脑图"任务一起做)。

## 8. 护栏
- 队列上限 + 背压;幂等键去重;steering 仅安全点;cancel 要干净(不留半态);pause 不丢任务;事件全程落盘可追溯。
