# 控制平面 · routing/(声明式 · 零 token 编排)

> 把"排队、打断、引导、路由"抽成**独立于模型**的一层。编排器只算条件表达式、选下一节点,**本身不调模型、不烧 token**。
> 依据蓝图 `shared/reference/多智能体架构设计.md` §5(多模型)、§8(控制平面)、§18(声明式控制平面)。

## 三条规矩(落到节点级强制)

1. **队列两语义**(§8):`steering`(打断式,工具执行后/下次 LLM 调用前送达)/ `follow-up`(排队式,当前任务干完才送达)。默认 `one-at-a-time`。
2. **上下文从严**(§18.3):`explicit`(只看指定依赖,**默认**)/ `last_only`(流水线)/ `accumulate`(全量,仅总管级)。
3. **何时交回 LLM**:仅当没有预定义边匹配 / 需临场拆新任务时,调一次 `planner` 节点重规划,产出子图后交回确定性编排。

## 三道护栏(§18.7)

- **防失控**:每条流程 `max_loops` + 墙钟超时,超限强停 + 报 human gate。
- **运行前校验**:`validate_before_run` 静态查路由可达(无死节点/死循环)、引用完整、表达式可解析。
- **定期回归**:pass^5(同组任务连跑 5 次全过率),验证"文件态记忆"是否更稳更省。
