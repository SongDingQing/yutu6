# 玉兔6 · 四智能体自优化循环(架构 v2)

> 给总管审核。审核通过后由总管"发布"(停旧循环、按本架构启动),并作为**常驻自优化任务**:今后老板给总管"优化/自优化/挑错 UI"类任务,总管自映射到本循环。
> 更新:2026-06-18。范围:玉兔6 WebUI(`projects/控制台/public/`)。只动 UI 层,不碰业务/密钥,Starlaid 不涉及。

## 0. 一句话
四个智能体**生产者-消费者解耦**协作,把 WebUI 越用越好用:优化师动脑,Peekaboo 当眼和手,开发(异步队列)落地,Hermes 报信。**按次数跑 10 轮**(不按定时),每轮跑到完成为止。

## 1. 四个智能体(职责分离)
| 智能体 | 角色 | 干什么 | 用什么 |
|---|---|---|---|
| **优化师** | `ui_optimizer`(Codex) | 读截图挑错(交互流畅性/易读性)+ 出**测试路线** + 评测试 + (隐式)回归 | codex(读 PNG/源码) |
| **Peekaboo** | `gui_desktop_control` | 截图(纯捕获,无需模型)+ 按测试路线**测试点击** | peekaboo(image + click/agent) |
| **开发** | `worker_code`(Codex) | **异步消费队列**,按问题改 `public/`;**不汇报、不阻塞循环** | Codex 主力,确保真实文件读写与测试;GLM 仅做低风险草案/摘要 |
| **Hermes** | 飞书信使 | 改前预告卡 + 收尾结果卡(简洁·有标题·有格式) | notify-feishu.sh |

## 2. 跑法:按次数 10 轮(不按定时)
**为什么改**:每轮"挑错+测试截图"耗时不定,3 分钟定时会把没跑完的轮截断、互相打架。改为**计数驱动**:跑满 `MAX_ITER=10` 轮(或提前收敛),**每轮跑到完成才进下一轮**,不被时钟切。

**生产者(优化师+Peekaboo)· 每轮**
```
① 复用单一验证标签打开页面 → Peekaboo 截图(改前) → shot_before.png
② 优化师读图 → 挑错 + 测试路线
③ Peekaboo 执行测试路线(点击/输入)→ 截后续态
④ 优化师评测试 → 合并最终问题清单(去重)
⑤ Hermes 飞书 → 改前预告卡
⑥ 把"修复任务"入队(`queue/*.task`),**不等开发**,直接进下一页/下一轮
```
**消费者(开发 worker_code)· 后台独立**
```
持续从 `queue/*.task` 原子领取任务 → 最小改动修 public/ → 归档到 `queue.done/` → 记日志(不回报循环)
```
> **隐式回归**:生产者每轮重新截图,自然看到开发已落地的改动 —— 无需单独阻塞的回归步骤。新发现继续入队,像队列一样累积。

## 3. 防卡死 · 边缘条件(硬护栏)
1. **每步超时**:每个 codex/peekaboo/worker_code 调用都包 `timeout`(codex 120s、peekaboo 60s、worker_code 300s);超时即跳过该步、记日志,**绝不挂起**。
2. **总轮数上限**:`MAX_ITER=10`,到顶停。
3. **早停收敛**:连续 2 轮去重后**无新问题** → 判定已优化到位,提前结束。
4. **问题去重**:对每条问题取指纹(hash),已入队的同一问题不重复入队,防队列无限堆。
5. **队列上限 + 背压**:pending > `QUEUE_CAP`(默认 12)时,生产者**暂停入队、等消费者消化**,防爆。
6. **单写锁**:开发同一时刻只跑一个(lockdir/flock),避免并发改同一文件打架(单写主原则)。
7. **失败跳过**:任一步失败/空输出 → 跳过本页本轮,不重试到死。
8. **收尾截止**:生产者跑完后给消费者**排空截止**(默认再等 300s);超时仍有剩余则记录未完任务、收尾,不无限等。
9. **互斥旧循环**:启动前检测并停掉已有 loop/dev-worker,**不双开**(双开会两批改同文件)。
10. **只读/写分离**:截图、挑错、测试都是**只读**,可与开发的写并行;只有 codex 改文件这一步串行加锁。
11. **验证标签不堆积**:打开 `/workspace`、`/control-room` 等验证页面必须走 `open-validation-tab.sh`,先查 Safari/Chrome 已有同页签;有则激活并刷新且关闭重复页签,没有才新开一个。严禁在循环里直接反复 `peekaboo open`/`open URL` 导致 workspace 标签增长。

## 4. 高效多智能体协同
- **生产者-消费者解耦**:优化师不等开发改完,产出即入队,开发异步消化 → 吞吐高、不互相等。
- **流水线并行**:优化师在看下一页/下一轮时,开发在改上一批 → 重叠执行。
- **写串行、读并行**:只有"改文件"加锁串行;看图/测试/挑错并行跑。
- **批量少打扰**:同一轮多页结果合并一张飞书结果卡。
- **方向基调不变**:仍只盯**交互流畅性 + 易读性**,增量小步(每条≤6 项),零依赖。
