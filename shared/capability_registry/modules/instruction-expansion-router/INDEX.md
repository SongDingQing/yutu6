# instruction-expansion-router

前门指令补齐器只作用于用户进入系统的第一跳。内部智能体接收结构化任务信封，不重复补齐。

## 作用

把口语、简略、截图或跨项目请求补成可独立执行的任务信封，明确目标、边界、输入、验收和待确认事项。

## 路由优先级

1. 用户显式点名的能力或 agent。
2. 当前项目已安装的项目命令补齐器。
3. 通用 wrapper。
4. 本模块的全局兜底。

每次任务最多产生一个可见补齐稿。项目能力包未安装时，不得借用其他项目的流程；使用通用模板并标记缺失能力。

## 绑定

默认前门 runner 为 Codex。秘书负责补齐与背景整理，再交 CEO 决策，不直接替代下游实施。
