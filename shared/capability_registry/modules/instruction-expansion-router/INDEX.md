# instruction-expansion-router(补齐指令稿 · 前门专用)

> **新机重建**的前门指令补齐路由器。只绑**前门总管 = Codex CLI**;内部 agent 不挂。
> 读序:本文件 → `expansion-spec.md`。

## 它解决什么
人对前门总管说话往往是**口语 / 简略 / 带截图 / 跨项目**的(例:"把那个修一下""做完发我")。直接派给下游会丢边界、丢验收、丢输入路径。本路由器在派单**之前**,把这类指令补全成 §4 的完整 **task 信封**(目标/边界/输入/验收/截止),并标注用了哪条补齐来源、置信度、还缺什么。

## 为什么只绑前门
- **人↔系统边界**才有"未补全的口语指令";越过前门后,层间走的是精确 task.md/result.md,已是补全态。
- 内部 agent 再挂补齐会**重复输出、上下文串味、浪费 token**(§3 need-to-know)。
- 所以:前门总管(Codex CLI)加载本规范进 L1 稳定前缀(§18.5);主管/员工不加载。

## 分发优先级(每次任务最多输出一个可见"补齐稿"块)
1. 用户**显式点名**的 skill/agent —— 最高,直接用,不再路由。
2. **项目专用 expander**(Simulaid / YuanXiao / Zongzi)—— 命中某项目就用它的补齐稿。
3. **跨项目 wrapper**(玉龙/玉灵/玉豚/玉凤/玉鼠/玉虎/玉衡/玉凰 等交付/内容/质量类)。
4. **全局 router**(本模块)—— 兜底,只在 1–3 都不命中时补;**只分发,不与上层重复输出**。

> 项目 expander / 跨项目 wrapper 多在旧机 41 技能里,**按需拉**(做到该项目时再拉,见 registry)。在它们到位前,前门由本全局 router 兜底补齐。
