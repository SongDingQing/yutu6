# 工程部门共享知识区

更新:2026-06-22

用途:承接 Codex 工程岗位切到 GLM-5.2 后的岗位知识、上下文和定位路径。接手方先读本索引,再读自己的岗位交接文档,最后按任务信封读源码。

## 接手顺序

1. 读当前任务信封、项目边界和验收。
2. 读 `shared/DATA-MAP.md` 中本岗位的知识定位。
3. 读本目录对应岗位交接:
   - `worker-code-handoff.md`:后端程序员 `worker_code`。
   - `it-engineer-handoff.md`:IT 工程师 `it_engineer`。
   - 前端工程师 `frontend_designer` 继续读 `projects/控制台/artifacts/frontend-handover.md`。
4. 再读岗位 `agent.json.read_paths` 中的项目文件。

## 最低交接格式

每份岗位交接必须包含以下五项,缺一项即视为交接未完成:

- 在研任务:当前或近期可能接续的任务、任务 id、产物位置。
- 已知坑位:容易误判、假完成、权限或工具 gate、测试限制。
- 知识库定位:DATA-MAP、agent 合约、项目文档、关键源码和历史 artifact。
- 上下游协作约定:主管、架构、前端、IT、维修、HR、质量运营之间怎么交接。
- 历史决策指针:为什么这样设计、哪些决策不能轻易反转、回滚位置。

## 可理解性门槛

- 交接文档必须写"为什么"和"何时用",不能只列路径。
- 每个岗位至少给出 3 个任务前必读入口和 3 个验证命令。
- 新接手 GLM-5.2 在首次独立接单前,由主管或 quality_ops 随机抽 3 条交接要点做回测:接手方需复述含义并指出文档定位,3/3 命中才算生效。

## 质量门槛

- 切换前基线来自 `projects/控制台/artifacts/model-cost-optimization-20260622.md` 与本轮迁移报告。
- 切换后按相同口径抽检每个切换 role 至少 20 个样本;样本不足时只允许继续观察,不得宣称全量质量稳定。
- 持平阈值:一次通过率 >=95%,重试率 <=5%,review 均分不低于切换前 baseline 的 95%,fallback/限流错误率 <=5%。
