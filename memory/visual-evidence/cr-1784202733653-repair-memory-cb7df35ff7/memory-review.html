可泛化维修经验

问题模式

高竞争文件资源锁缺公平授予时，合法长持锁与后到抢占会使旧 waiter 饥饿到超时；worker 仍存活，却可能被误报为 worker crash。

根因

授予只看当前 holders；轮询重写 requested_at 使顺序漂移；wait 审计无节流、公共写域过宽、失败类型未分账又会放大并掩盖问题。

解法 / 预防

稳定 requested_at + token，对冲突 waiter 实施有界公平；按状态或时窗节流审计；缩小写域；将 resource-lock-timeout 作为独立 failureKind。
