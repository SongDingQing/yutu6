# 玉兔6 多智能体编排系统 · 完整复刻文档

> 目标:让另一个工程师据此**直接复刻**整套系统。技术细节均逐文件核实。日期 2026-06-20。
> 一句话定位:**本地、零依赖(Node 原生)、"文件系统即真相"的多智能体编排系统**——老板只对秘书说话,秘书补全→CEO 拆解路由→员工执行→校验→完成,全程事件可追、崩溃可恢复。运行在一台 Mac mini。

---

## 0. 设计哲学(复刻前先理解)
1. **文件系统即真相**:队列、事件、任务态、记忆全是磁盘上的 JSON/JSONL/MD 文件,无数据库依赖;任何时刻 `cat` 文件即可知道系统状态;进程崩了重启即恢复。
2. **零运行时依赖**:核心引擎纯 Node 原生(无 Express/无框架),只在边缘用可选 CLI(claude/codex/peekaboo/hermes)与 Python(meowa 生图、knowledge RAG)。
3. **声明式编排**:角色→模型、角色→runner、流程图都是 YAML/JSON 声明(`config.json` + `shared/routing/*.yaml`),路由零 token。
4. **组织隐喻**:用"公司"建模——董事长(老板)→秘书→CEO→项目主管→员工+外围专家;UI 是等距像素办公室 + ComfyUI 风链路图。
5. **need-to-know 上下文**:每个 agent 只读白名单路径(`agent.json.read_paths`),信息从严。

## 1. 创新点(与主流多智能体框架的不同)
- **文件队列 + 原子写 + 三层串行锁**:不用消息队列/数据库,用目录状态机(`queued/running/paused/done/failed/canceled`)+ `temp+rename` 原子写;崩溃可恢复、可审计。
- **声明式角色路由 + 多模型兜底链**:订阅(claude/codex)→ API(anthropic/openai/zhipu)→ 本地(ollama)三级 failover,余量预测性切换;省钱(免费/买断额度优先)。
- **自愈维修员闭环**:任务失败→指纹去重+冷却→自动开维修工单→特权 Codex 修→写根因+验证+知识沉淀,无需人工。
- **五层长期记忆 + RAG**:`memory/`(偏好/决策/经验/实体/会话)提炼注入 + `knowledge/` 原文检索;系统"越用越懂老板、越少踩重复坑"。
- **桌面控制 + 动作截图硬验收**:Peekaboo(眼,截图不接模型)→ Claude(脑,读图挑错)→ Codex(手,改代码);GUI 动作前后截图比对、未落地自动纠错。
- **组织隐喻可视化 UI**:等距像素办公室 + 链路图,把抽象的 agent 协作变成"看得见的公司在干活"。
- **人机接口角色化**:秘书 = 老板的唯一接口(传声筒 + 验证者),CEO = 经验大脑(拆解/分配),职责边界清晰。

## 6. 队列引擎 + 一个任务的全流程

### 6.1 队列(`shared/engine/queue.js`)
- 每 agent 一队列 `queues/<agent>/`;文件名 `PP-SSSSSSSSSS-<id>.json`(PP=优先级00最急,S=10位序);**字典序=取用顺序**。
- 原子写:temp(`wx`)+rename。
- API:enqueue / list / setPriority / jump(=setPriority 0) / reorder(拖拽重排) / steer(运行中注入引导) / pause / resume / requeue(重试) / cancel / claim(原子取队首→running) / finish(终态)。

### 6.2 三层串行锁(`ceo-worker.js`,`ENGINE_MAX_CONCURRENCY=1`)
- engine slot(全局并发=1)+ runner-type single-flight(claude 一次一个)+ active-ceo-task 锁(一个 CEO 任务的整条派生链跑完才放下一个)。全部 PID+心跳(2s 续约)判活;孤儿 `sweepStaleRunning`(5s)按 PID 存活重入队(retry≤2)。

### 6.3 一个任务的完整生命周期(事件序列)
```
1. 老板在 webUI 给"秘书"派单
   → POST /api/queue/ceo (或 secretary)
2. 秘书(handleSecretary):补全指令为完整信封 → secretary.expanded → 入队 ceo
3. ceo-worker claim → 等 active-ceo-task 锁 → spawn engine-runner(flow=project-route)
   → task.created → node.start orchestrator-plan
4. CEO 规划:claude 输出 {orchestrator:{projectId,summary,acceptance}}
   → inferProjectId 判归属(① 显式projectId最优先 ② 关键词规则 ③ 兜底默认)
   → 判不准则 project.route.paused(软暂停,退码5,转老板确认,不杀)
5. project.brief.written(写 projects/<id>/brief.md) → project.routed
   → queue.enqueued 到 supervisor-<项目>(flow=review-loop)
6. supervisor-worker claim → engine-runner review-loop:
   node.start implement(codex 改代码,输出 {implementation:{done,changed_files}})
   → node.start review(主管/claude 输出 {review:{pass,severity}})
   → pass 则 node.end → task.done;fail 则回 implement(重试≤2)
7. 完成状态向上传递:子任务真完成 → 父任务 done;子失败 → 父 failed
   (修复后,不再"路由完就假装完成")
8. supervisor 完成 → notifyProjectDone → 飞书通知老板
9. 失败任意环节 → openAutoRepairTicket(指纹去重+30min冷却)
   → 维修工单入队 repair(priority 0)→ 维修员修 → repair.ticket.completed
   → 记忆官提炼经验进 memory/experience.md
10. 秘书验收 → 报告老板
```

## 7. 记忆系统 + 知识库

**`memory/` 五层(Markdown,记忆官 + 秘书维护)**:
- `preferences.md` 老板画像(语言/风格/红线/成本观/审美/协作习惯)
- `decisions.md` 决策记忆(定了什么+为什么)
- `experience.md` 经验库(成功模式 + 失败教训:现象→根因→做法)
- `entities.md` 实体记忆(项目/角色/能力/机器/外部案例)
- `sessions/` 会话摘要
- `INDEX.md` 索引 + 5 条规则(提炼非流水账/条件→结论/去重纠正/对事不对人/密钥+Starlaid 绝不写入)

**记忆官三机制**:主动提炼(task.done / repair-ticket-complete / 会话结束)、检索注入(按相关性注入 agent 上下文)、定期整合(consolidate)。维修工单完成的"知识沉淀候选"→ 泛化模式进 experience、项目↔技术↔路径进 entities。

**与 knowledge/(RAG)分工**:memory/=提炼后快速注入;knowledge/=原文检索(`ingest.py` 扫 wiki→分块嵌入→`kb.sqlite`;`query.py` 向量+FTS5 融合+图谱扩展)。维修结果不直接写 kb.sqlite。⚠️ **RAG 已建好但 agent 尚未接入检索(待补)**。

**密钥**(全部本机 env 文件,绝不进源码):`artifacts/new-api/internal-token.env`(NEW_API_TOKEN)、`artifacts/new-api/zhipu.env`(ZHIPU_API_KEY)、`~/.hermes/.env`(FEISHU_APP_ID/SECRET/HOME_CHANNEL、MEOWART_API_KEY)、`~/.ssh/id_ed25519_gitee_yutu6`。
**启动**:① 配 `config.json`(runners + roleRouting)与 `shared/routing/*.yaml`;② `node projects/控制台/server.js`(监听 41218,自动拉起 ceo-worker 与 backlog 监督);③ 打开 `http://127.0.0.1:41218/workspace`;④ 入队即跑。
**守护**:launchd `com.yutu6.console`(KeepAlive+RunAtLoad);重启 `launchctl kickstart -k gui/$(id -u)/com.yutu6.console`(改了引擎/config 必须重启才生效——这是踩过的坑)。
**可移植内核**(换机/复刻只需这些):`shared/engine/*` + `config.json` + `shared/routing/*.yaml` + `shared/agents/*`;控制台三件套 `server.js`+`ceo-worker.js`+`engine-runner.js`。
