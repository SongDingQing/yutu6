# 玉兔6 通用系统契约

## 目标

玉兔6 是文件式多智能体编排系统。公共发行版只提供系统部门、执行引擎、队列、工作区和项目部门模板；业务能力通过独立项目能力包安装。

## 固定组织链路

1. 董事长在工作区提交目标。
2. 秘书补齐背景、边界、验收与项目归属。
3. 需要事前评审的任务先经董事会，再由 CEO 决定路线。
4. CEO 写项目 brief，并交给 `supervisor-<projectId>`。
5. 项目主管派执行员工、复核证据并更新项目状态。
6. done gate 只有在逻辑链、真实证据和复审同时通过时才允许完成。

维修与系统救火走维修部门，不占普通项目执行槽。

## 数据边界

- `board/`:方向、拍板和趋势。
- `projects/<projectId>/`:项目 brief、状态、任务和产物。
- `shared/`:系统引擎、角色、路由和通用能力。
- `memory/`:当前安装实例提炼后的长期记忆。
- `~/.config/yutu6/`:API key、provider 配置、PID 和日志等本机私有运行数据。

密钥、登录信息、原始会话和业务私有资料不得提交到公共仓库。

## 项目扩展

新业务必须先通过项目部门生成器创建。项目专属 agent、skill、参考资料和密钥映射放在独立项目能力包中，安装时登记到 capability registry；系统核心不得反向依赖任何具体项目。

## 首次启动门禁

配置向导必须确认至少一个可执行 CLI 和一个 API 模型连通。检测失败时保留在 `/setup`，通过后才开放工作区。CLI 登录由用户在供应商官方命令中完成，网页不收集 CLI 密码。
