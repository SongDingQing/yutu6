# 办公室像素场景 · 系统设计 + 动作序列(分步落地·严控成本)

> 秘书出规格。核心约束(老板):**完全用图块拼接、画风统一、动作一个一个做、避免白花钱(meowa 生图/动画花钱)**。出规格 2026-06-19。

## 0. 现有素材(优先复用,别重复生成!)
- **等距 seamless 地块**:`office-floor-seamless-isometric.png`、`office-wall-seamless-isometric.png`(可无缝拼)。
- **各角色坐姿 sprite(idle/working)**:`sprite-seated-{ceo,secretary,supervisor,worker,outsourcer,edge}-{idle,working}.png`。
- **董事长坐姿动画**:`chairman-idle.webp` / `chairman-working.webp`。
- 董事长办公室家具/props:`chairman-office-*`、`prop-00~05`。

## 1. 问题(对应老板反馈)
1. 地图没真用图块拼:tile 当 CSS background、`background-size` 与画板尺寸不对齐 → 图块数量对不上、错位、大片空地。
2. 总裁办公室用大贴图(lounge/workzone)而非图块拼。
3. 角色没坐进工位、画风粗糙、桌后乱放 props。

## 2. 目标:完全图块拼接的连续等距办公室
- **地块**:等距菱形网格,整数个 tile 无缝拼接、数量对齐、铺满每片区(用现有 seamless tile,修正网格对齐算法,别用 background 硬铺)。
- **角色**:用现有 `sprite-seated-*` 坐在各自工位椅子上;层级 z-order = 地块→家具→角色;**绝不遮挡**场景。
- **props**:每工位只留 桌+椅+电脑;删掉桌后莫名道具。
- **画风**:统一像素规格;粗糙的角色**逐个评估**是否重生成(控成本,别一次性全重生)。

## 3. 动作序列(董事长发命令 → 一个一个做,做一个验一个)
> 老板发命令 = 派单事件触发这套连贯动画。**优先 CSS/transform + 现有 sprite,新 meowa 仅在近似不够时逐个补。**
- **步骤①** 董事长坐姿 idle:复用 `chairman-idle.webp`(坐办公室椅)。**不花钱**。
- **步骤②** 发命令→董事长把"指令稿"放到桌上:指令稿=小纸张 prop,CSS 在董事长桌面淡入/滑出。(prop 可先用色块/简图,**不花钱**;要精致再 meowa 出一张。)
- **步骤③** 秘书起身走到董事长桌拿稿:CSS 位移现有秘书 sprite 过去 + 拿稿。(走路姿势先用 idle sprite 平移近似,**不花钱**;要走路帧再 meowa。)
- **步骤④** 秘书回工位、打字输入:切 `sprite-seated-secretary-working`。**不花钱**。
- **步骤⑤** 邮件飞到 CEO 电脑:邮件图标 CSS 从秘书电脑飞到 CEO 电脑屏。**不花钱**。

## 4. 成本控制(硬原则)
优先级:**复用现有素材 > CSS/transform 动效 > 新 meowa(逐个、做一个验一个、绝不批量)**。每次只做一个动作/一类素材,截图给老板确认连贯性后再做下一个。

- **第一步(本次·不花钱)**:地图图块正确拼接 + 董事长坐姿(chairman-idle.webp)+ 各角色坐工位(sprite-seated-*)+ 清 props。全用现有素材 + 前端逻辑。验收看:图块无缝拼、数量对齐、角色坐工位不遮挡、props 干净。
