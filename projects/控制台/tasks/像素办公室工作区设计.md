# 像素办公室工作区 · 设计方案

> 秘书出方案(老板审阅)→ 定方向后拆「meowa 素材」+「codex 前端」两条落地。出方案 2026-06-19。
> 目标:用 meowa 的「人物动画 + 地图图块」做一个像素风办公室工作区,比通用方案精致;agent = 带动画的像素小人,按组织层级坐工位,状态/交接由真实数据驱动。

## 1. 概念
- 工作区新增**第三视图「像素办公室」**(与现有「工位」「链路图」并列,复用 #22 双视图的切换框架)。
- 一个像素风办公室场景:每个 agent 是一个**带动画的像素小人**,坐在按组织层级布置的工位;待命/工作/完成/失败用小人动作 + 头顶气泡体现;交接(`edge.take`)用小人之间的传递动效。
- **精致来源**:素材全部用 meowa 生成(角色多状态动画 + 等距办公室 tileset),真实数据驱动,不是静态贴图。

## 2. 视觉风格(待定 → 见结尾选项)
- 视角:**等距 isometric(推荐,最像精致办公室游戏)** / 俯视 top-down(像 AI 小镇)/ 横版 side。
- 调色:贴现有深色控制台(深底 + 角色 accent);像素分辨率统一(如 32px/64px sprite)。
- 角色形象:沿用现有标识做像素化(CEO 兔子、秘书信封、设计师调色板、Peekaboo 显示器…),保持辨识度。

## 3. 布局(对应组织层级,见 mockup)
- **董事长办公室**(顶层独立间):董事长(预留位 / 指令入口)+ 秘书。
- **CEO 办公室**。
- **项目主管区**(开放工位,按项目分「岛」):控制台岛、Simulaid 岛(主管 + 员工),随项目动态增减。
- **外围区**:质量运营 / 监管 / Peekaboo / 优化师 / Hermes / 智谱设计师。

## 4. meowa 素材清单(用已接入的 meowa 统一能力生成)
- **地图**:办公室 tileset(地板/地毯/墙/隔断/门)+(若等距)isometric 地图 tile —— `tileset-gen-run` / `isometric-gen-run`。
- **家具 props**:办公桌、椅、电脑、显示器、白板、植物、文件柜、董事长间沙发、会议桌 —— `pixel-gen-run`(object/props 模板)。
- **角色**:每个 agent 一套像素小人 + 动画 —— `pixel-gen-run`(character)+ `animate-run`,输出 spritesheet/WebP:
  - idle(待命:呼吸/打盹)、working(敲键盘)、done(✓/小庆祝)、fail(低头/红气泡)。
  - 可选 `character-multi-view-run` 出多方向,便于交接走动。
- **UI**:状态气泡(工作中/完成/失败)、任务小牌 —— meowa 或 CSS 皆可。
- 产物存 `projects/控制台/artifacts/office-assets/{tilesets,props,characters,anim}/` + 一份素材清单 json(名称→文件→用途)。

## 5. 技术实现(前端,codex)
- 第三视图「办公室」tab(复用 #22 切换 + localStorage 记忆)。
- 渲染:canvas(或 DOM+CSS sprite)——铺 tileset 场景 + props + 角色 sprite 定位 + 帧动画;等距用 isometric 投影。
- **数据绑定(复用现有数据层,零新接口)**:
  - `agents[role].state` → 角色动画状态(idle/working/done/fail)+ 头顶气泡。
  - `agents[role].curTask` → 工位任务小牌。
  - `edge.take` 事件 → 小人之间交接动效(文件飞过 / 走动 / 高亮路径)。
  - 项目清单 → 项目岛动态生成(对齐项目制编排)。
- 性能:sprite 图集 + `requestAnimationFrame` + 仅可见时播放;素材懒加载。

## 6. 分阶段计划
- **阶段 0 · 美术风格稿**:智谱设计师(GLM-5.2)出风格指南 + 布局图 + 角色设定 + 动画规范;秘书给初稿方向。
- **阶段 1 · meowa 批量生成素材**:按风格稿,meowa 生成 tileset/props/角色/动画,建素材库 + 清单(智谱设计师 / Simulaid 美术 用 meowa 能力执行)。
- **阶段 2 · 前端办公室视图**:codex 做第三视图 canvas 渲染 + 数据绑定。
- **阶段 3 · 动效打磨 + 验收**:工作/交接/完成动效、状态气泡、性能;Peekaboo 截图对照。

## 7. 比"马维斯"更精致的点
- meowa 高质量像素素材(角色多状态动画 + 等距办公室),非通用占位图。
- **真实数据驱动**:状态、任务、交接都来自引擎事件,不是装饰动画。
- 组织层级清晰的办公室布局(董事长办公室 → CEO → 项目岛 → 外围),随项目动态生长。
- 流畅交接动效 + 头顶状态气泡,一眼看懂"谁在干啥、交给谁"。
