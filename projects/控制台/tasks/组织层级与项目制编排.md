# 组织层级落地:秘书接入 + 董事长办公室工位 + 项目制编排 · 设计规格(给 codex)

> 流程:秘书出规格 → 老板转交 codex → 秘书验证。出规格 2026-06-19。
> 依赖:**「稳定地基」(多队列常驻监督/并发/孤儿清理)先落地**;本任务排其后(复用其多队列与 worker 机制)。

## 0. 目标层级 & 匹配度结论
目标:**董事长(老板)→ 秘书 → CEO → 项目主管(一项目一个)→ 员工 + 外围**。

**这与系统原设计高度一致**(无需推翻,只需显式化 + 把单流程升级为多项目编排):
- `shared/agents/orchestrator/prompt.md` 已写明 CEO 职责:"接指令(前门 `instruction-expansion-router` 补全)→ 拆 `board/direction.md` 到各项目 `brief.md` → 派给对应主管 → 读各 `status.md` 增量更新 `board/status-rollup.md`"。
- `shared/agents/supervisor/agent.json` 已定义"**项目主管(一项目一个)**,`scoped_to_project`,只看自己项目"。
- 数据结构已就位:`board/direction.md`、`board/status-rollup.md`、`projects/<p>/brief.md`、`projects/<p>/status.md`、`projects/<p>/tasks/`。
- projects/ 下确有多项目:Simulaid、控制台(排除 `_` 前缀的模板/迁移)。

**差距(本任务补齐):** ①董事长/秘书无显式工位、秘书未接入 agent 体系;②引擎实际跑的是单 `review-loop`(单 supervisor),未按项目动态分派到项目主管实例。

## 模块 1 · 秘书接入(前台 Cowork + 后台 runner)
- 加 `secretary` 角色:`config.json` roleRouting 增一条(label「秘书」,runner `claude`);`shared/agents/secretary/{agent.json, prompt.md}`。
- 秘书职责(prompt):接老板指令(前门)→ 经 `instruction-expansion-router` 补成完整 task 信封 → 派给 CEO(入 `ceo` 队列)。只路由/补全/监督,不碰项目细节。红线同 CEO(密钥只本机/登录交主人/Starlaid 排除/拿不准就停)。
- 秘书队列 `queues/secretary/`:老板指令可进秘书队列由后台 secretary worker 自动处理;**同时保留前台 = 本 Cowork 会话(我)**直接对话/派单(双腿,互不冲突)。
- 复用稳定地基的通用 worker + 常驻监督。

## 模块 2 · 项目制编排(引擎核心,复用 #18 多队列)
> #18 已实现"任意 agent 一条队列 + 通用 worker",多项目主管几乎现成;主要改 CEO 的"选项目 + 路由"。

- **项目注册**:扫 `projects/*`(排除 `_` 前缀)得项目清单;每个项目对应 `supervisor-<project>` 队列与 worker。
- **CEO 选项目**:`engine-runner.js` 的 `orchestrator-plan` 节点输出 JSON 增加 `projectId`(CEO 判断任务属哪个项目;老板/秘书也可在派单时显式带 `projectId`)。
- **派到项目主管**:CEO 把 brief 写 `projects/<projectId>/brief.md`,并入队到 `queues/supervisor-<projectId>/`(用 `Q.enqueue`),而不是直接进固定 review 节点。
- **项目主管实例**:为有活的项目起 `ceo-worker.js --agent supervisor-<projectId>`(通用 worker,`scoped_to_project=<projectId>`);它跑该项目的 `review-loop`(implement→review→done,scope 限定该项目)。
- **回流**:项目主管完成 → 写 `projects/<projectId>/status.md`;CEO 读各 `status.md` **增量更新** `board/status-rollup.md`(严禁整篇重写)。
- **兼容**:不带 `projectId` 的任务(如控制台自身改造)默认 `projectId=控制台`,保持现有 `review-loop` 能跑、不破存量。
