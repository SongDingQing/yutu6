# 任务看板 Task Board · 设计规格

> 秘书出规格。目标:把工作区右侧「队列区」重做成一个清晰的任务看板,老板派完任务点开网页就能一眼看到进度。出规格 2026-06-19。先于办公室动作序列做。

## 区域结构(右侧)
1. **进行中**(顶部):正在处理的任务卡 —— 每张显示:执行的**智能体**(头像 + 名)+ **任务摘要**(一行,≤一句)+ **处理时长**(从 `started_at` 实时计时,如 `2分15秒`)。让老板一眼看出"任务在进行"。
2. **待办 / 备选**(右下):**洞察员推荐** + 老板**手动「回来处理」**的任务,作为备选;点「启用」才入队执行。(复用公告板 `cards.json` 的 todo,`source` 区分 洞察员 / 手动。)
3. **过往**(底部):已完成的任务,**划掉 / 置灰**(参考 Progress 已完成样式),保留**最近 10 个**,旧的挤掉(滚出)。

## 上限
- 进行中 + 待办 **≤ 20 个**;过往队列 **≤ 10 个**(超出旧的滚出)。

## 数据来源(复用现有,零新接口)
- 进行中 = 各 agent 队列 `running`(用 `started_at` 算实时时长)。
- 待办/备选 = 各 agent `queued` + 公告板 `cards.json`(todo,`source`=洞察员/手动)。
- 过往 = 各 agent `done`(按完成时间取最近 10)。
- 复用 `/api/queue/:agent` + `/api/bulletin` + `engine-events`;沿用 1.5s 轮询刷新。

## 效果
老板派任务 → 点开网页 → 任务板「进行中」立刻看到(谁在做 / 什么任务 / 跑了多久);备选在右下;完成的置灰滚动。清晰看出"任务已加进来、在跑"。
