# 待办公告板 Bulletin Board · 设计规格(给 codex)

> 流程:秘书出规格 → 老板转交 codex → 秘书验证。出规格 2026-06-19。
> 排在「双视图 #22 + 组织层级工位」之后(同改 `public/workspace.html`,避免抢文件)。

## 目标
工作区右侧加一块「**待办公告板**」:暂存候选/待办任务卡,**点击「启用」即把该任务入队执行**。
与「队列」区别:队列 = 已派发、在跑/排队;公告板 = **暂存池,点击才派发**。老板/秘书可往板上放任务(记着不立即跑),需要时一点启用。

## 数据
- 存 `projects/控制台/artifacts/bulletin/cards.json`:数组,每卡:
  `{ id, title, desc, target(队列 agent,如 ceo / zhipu-designer / supervisor-<project>), project?, payload(完整 task 信封,= 入队用的 task 对象), status:'todo'|'enabled', created_at, enabled_at? }`。
- 持久化到文件;并发安全(读改写加简单锁或原子写)。

## API(server.js)
- `GET /api/bulletin` → 列所有卡。
- `POST /api/bulletin` → 加卡(body 含 title/desc/target/payload)。
- `POST /api/bulletin/:id/enable` → 用该卡 `payload` 调现有入队逻辑(`Q.enqueue` 到 `target` 队列 + `ensureQueueWorker`),卡 `status='enabled'`、记 `enabled_at`;返回入队结果。
- `POST /api/bulletin/:id/remove` → 删卡。
- 复用 #18 多队列 + 稳定地基(入队后被常驻监督拉起执行)。

## 前端(workspace.html 右侧)
- 右侧栏加「待办公告板」板块(**bulletin board / 便签卡片风**:轻拟物、卡片/便签、可区分 todo / 已启用)。
- 每张卡:标题 + 描述 + 目标 agent 标签 + 「启用」按钮(+「移除」)。
- 点「启用」→ `POST enable` → 入队;卡变「已启用」(灰掉或移到底部)。
- 加卡入口(简单表单:标题/描述/目标/可选 payload);老板/秘书可往板上放任务。
- 轮询 `GET /api/bulletin` 刷新(对齐现有 1.5s 轮询)。
