# 任务:工作区页 → 脑图化协作 + 修任务串台 + 状态监控

> 主管派给开发。**等 UI 自优化循环跑完再做**(避免和 dev-worker 抢 `public/`)。范围:`projects/控制台/public/workspace.html`。零依赖或仅 CDN(Mermaid 可用);localhost;不破现有派单/事件功能。

## 三个问题
1. **右下角事件流看不懂**:纯文字 `type 时间 - 文案` 日志,看不出"工作流在做什么、谁交接给谁"。
2. **没有交接的逻辑展示**:几个智能体之间的"派单/传递/汇报/交任务/确认"没有可视化。
3. **任务串台 bug**:某智能体(如主管)状态已变,但任务行显示成**上上次**的任务。
   - 根因:`applyEvent`(workspace.html 约 133–148)按 **role** 维度 `setDesk(role,state,taskText)`,任务行被任意后到事件覆盖;`taskText` 还会回退成 `ev.node/ev.type`;迟到/乱序的旧任务事件会盖掉新任务显示。状态与任务没绑在同一个 task 上。

## 要做
### A. 脑图/流程图化协作(替换或并入右侧"真实事件流")
- 用**节点图**(脑图/流程图,可用 Mermaid 或手写 SVG/HTML)展示智能体协作:
  - 节点 = 各智能体:总管 / 主管 / 员工(写码·窄任务)/ 质量运营 / 监管;以及自优化四件套:优化师 / Peekaboo / 开发 / Hermes。
  - 连线 = 交接;**连线上用文字标注动作**:`派单` / `传递` / `汇报` / `交任务` / `确认`(从 `edge.take` 的 from→to + 事件类型推)。
  - 实时反映:当前活跃节点高亮、连线显示最近一次交接方向与动作。
- 文字事件流可保留为"明细"小窗,但主视图是脑图。

### B. 修任务串台(核心)
- 改成**按 task 跟踪**而非按 role 覆盖:每个智能体维护"**当前任务**"(以最新 task 为准);
- **旧任务的迟到事件不得覆盖更新任务**(用 task 的 seq/时间判断,只让"最新 task"更新该智能体的显示);
- 任务结束/空闲要**清掉或标注**(如"已完成:X" / 回到"空闲"),不要残留上一个任务文案;
- `taskText` 优先用 `taskGoals[ev.task]`(任务目标),没有再退而求其次,别用 `ev.node` 冒充任务名。

### C. 状态监控(准确)
- 每个智能体显示:**状态**(空闲/工作中/完成/失败)+ **当前任务** + **最近交接**(从谁来 / 交给谁)。
- 状态机与事件严格对应:`node.start→工作中`、`node.end→完成`、`node.fail→失败`、`edge.take→交接`、`task.done/failed→收尾`;不同 task 的事件不串。
- 自优化循环的智能体状态可从 `projects/控制台/artifacts/ui-optimize/`(reports/queue/dev.log)读,或后端加一个事件源;先把引擎事件这套做对,自优化的接入可作第二步。
