# 等距办公室 · 游戏化愿景与文档库总纲

> 定稿人:老板(2026-07-07)。本文件是"等距办公室"文档库的**根**,统领 skill 与各细分规范。
> 每次生成/处理办公室美术前,先读本文件对齐愿景,再进具体规范,防注意力稀释。

## 一、愿景(老板原话锚定)
> "我想要的其实是**等距画面,看起来就像是游戏一样**。愿景是希望**后续所有办公室画面都像游戏**,我可以**随时启用过去的地图格子拼出一个新办公室,并添加新员工**,员工可以**通过动画离开工位去娱乐、或给领导汇报**等等。"

拆成 4 条不可动摇的核心约束(=skill 顶格锚点):
1. **等距(isometric 2:1)**——不是平面图、不是全景照片。
2. **格子化可拼装**——办公室 = 复用的 tile 拼出来的,不是一张整图。
3. **游戏化**——像模拟经营游戏,不是数据看板。
4. **员工是活的**——有工作/离位/娱乐/汇报等行为动画,不是贴图。

## 二、系统总览
```
等距坐标系(grid-spec V3: 128×64, pitch 64/32, z 分层)
  ├─ 地块层  floor/rug/edge/corner        ← isometric-gen / tileset-gen
  ├─ 家具层  工位2×2 / 董事长5×5 / 墙 / 道具 ← isometric-gen(footprint 完整)
  ├─ 角色层  9 类员工 sprite(坐/走/娱乐/汇报) ← pixel-gen + character-multi-view
  ├─ 动画层  循环动画(打字/行走/娱乐/汇报)   ← animate-run(spritesheet)
  └─ 拼装层  布局=格子数组 → 按 anchor 落格 + z 叠放 ← office-layout-schema
```
已落地基础:V3 网格规范 + 校验脚本 + 审批 ledger + 4 件 alpha 成品(地毯/空工位/董事长桌/绿植)。
**缺口(本次要补):** ①拼装地图数据格式 ②员工行为动画状态机 ③把成品接进 app(现在 app 还跑旧 CSS 占位)。

## 三、分期路线(★ 请老板圈定"近期做到哪一期")
| 期 | 目标 | 交付物 | 工程量 | 依赖 |
|---|---|---|---|---|
| **P1** 静态等距场景 | 一个董事长办公室,等距、有厚度、深色游戏风,**静态图** | 拼好的一屏 + 4-5 件补全 tile | 小 | 已有 alpha |
| **P2** 循环动画 | 员工在工位打字/董事长办公,**看起来在动**(原地循环,非移动) | 每角色 idle+working spritesheet,app AnimationDrawable 播 | 中 | P1 + 角色 sprite |
| **P3** 可拼装 | 从**预设布局**里选/换,或给指令拼新办公室、加员工 | office-layout-schema + 2-3 套预设 + 拼装渲染器 | 中 | P1 网格数学统一 |
| **P4** 员工行为 | 员工**离工位走去**娱乐区/领导处汇报,行为由真实 agent 状态驱动 | 八向行走 sprite + 状态机 + 寻路 + 状态→动画映射 | 大 | P2+P3 全 |
> 建议:**先做 P1 出一屏样板验证画风+管线**(走元宵采纳),对了再逐期加。避免一上来奔 P4。
> "拖拽编辑器"(app 内手动拖格子拼)属 P3 之上的**远期**,默认不做,先用"预设/指令拼"。

## 四、★ 待老板拍板(定了我才写深 layout-schema / 动画状态机)
1. **近期做到哪一期?**(建议先 P1;你若想直接铺到 P2/P3 告诉我。)
2. **"随时拼新办公室"的方式:** (a) 你在 app 里拖拽格子拼(大工程,远期) / (b) 你给指令或从预设里选(小,近期默认)。**默认按 (b) 设计**,认可请忽略,不认可请说。
3. **tile 尺寸裁决确认:** 两份旧文档打架,我裁决 **128×64(grid-spec V3)** 为唯一真相,废弃 192×126。如无异议按此推进。
4. **员工行为状态清单(P4 用):** 目前只有 idle/打字/秘书递稿。你愿景里的"娱乐""汇报"具体要哪几种?(如:喝咖啡/走动/开会/汇报/摸鱼)——到 P4 再细化,先记这里。
