## 4. 防复发规则(可执行、可验证)

### NR16(本窗新增,重大)· 断言仍红的「残余债」结案 = 假闭环,不计为已闭环
- 规则:任何 smoke/测试断言的失败,**不得**靠把它标注「残余测试债 / 与本次无关」就把工单 `status: done`;凡 daily 硬化复核检测到「某断言当前仍红」时,**必须把『仅有一张以残余债/不相关之名 done 的旧票』视同无有效工单(判红)**,强制要求一张新的专项工单(读全量断言 → 定位 → 归因 → 复跑绿后方可结案)。
- 与既有规则关系:这是 NR13「决策需执行消费者」在**结案标准侧**的补丁——堵住「决策要求开工单,于是有人翻出/新建一张 done 票充数,但根因仍未解」的旁路。是 [[沉淀≠落地]] / declared-but-not-honored 家族:**工单 status 字段声明「已解决」,但运行时断言事实仍红 = 声明与事实不符**。
- 验证:① 给定一条当前仍红的 smoke 断言,扫描其相关工单——若全部为 `done` 且无「该断言复跑绿」证据,硬化复核判红并要求新票;② mechanisms-smoke 专项检查:断言红 + 关联工单仅 auto-20260623051327(done,残余债)→ 应判「无有效工单」。

### NR14②(本窗提级:观察 → 执行)· 05:00 daily 惊群应错峰投递
- 升级依据:NR14②(06-27 立)原设「连续 ≥3 天 05:00 准点复发才判惊群」。现已**连续 2 个窗口、2 种不同症状**命中同一 05:00 同点火根因(06-27 `worker-heartbeat-stale`→看门狗重启;06-28 `resource.scheduler.all_blocked`→锁争用 ~3 分钟)。
