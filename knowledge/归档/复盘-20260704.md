## 防复发规则

1. Provider/runner 启动类失败先 incident 化:同一 provider/runner/scope 冷却窗内只允许一张 owner 票,后续任务 pause/backoff/waiting,不得刷高优维修票。
2. 主动 cancel 必须分账:用量审计、自动优化暂停、系统显式取消只产生 intentional cancel,不得计入 task.failed 或触发 repair.ticket.created。
3. all_blocked/watchdog restart 固定入复盘:all_blocked >100 或 restart >=3 必须列问题;事件缺 holder/reason/top domain 时,可观测性本身判未闭环。
4. 全局技能与 module registry 启动 smoke:无关 SKILL.md YAML 坏、registry 路径缺失都不得拖垮控制台任务;缺失时应清晰降级。
5. done gate 事件必须可解释:失败事件必须回指具体缺失 key、artifact 和 parse 状态;事件名不得掩盖实际可核证据。
6. board 决策每日同步 memory:当日 board 已决标题未进入 memory/decisions.md 时,governance 复盘必须追加并记录证据。
