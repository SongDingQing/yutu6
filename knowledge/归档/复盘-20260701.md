## 2. 根因分析

### P-1 · 自动证据行反向制造门禁义务

现象上,任务产物实际已有 a11y proposal-only 文档和视觉证据,但 review-loop 仍被 done-gate 打回「缺少 decisions.md 行号」。维修票复核确认,老板/CEO 需求从 ceo -> supervisor -> worker 未丢失,交接遗漏发生在机制层。

根因:`buildStructuredAcceptanceTable` 生成的自动“视觉/UI证据”行包含“对照设计”关键词;`validateStructuredAcceptanceTable` 原先把 requiredRows 全部拼进 designText,使这条自动证据行命中 `DESIGN_GATE_RE`,从而在验证阶段反向制造 `decisions.md:行号` 必填义务。生成表与验证表不一致,导致真实完成被假失败、重试、开维修票。

避免方法:自动生成的验收/证据行只能校验自身证据,不得作为新业务门禁触发源。设计门只看用户/上游显式 goal/acceptance/bounds 或非自动验收行;自动视觉行只校验截图/复核证据。维修已在 `shared/engine/done-gate.js` 排除自动视觉行触发源并补 `tests/done-gate.test.js` 回归,本复盘不再改代码。

### P-2 · provider 忙先被队列 retry 消耗,而不是在同节点 failover

现象上,`frontend_designer` brief 已完整进入 task.md,失败发生在 runner 工具启动/模型规划阶段;三次 run_attempt 都在 0.9 秒内因 zhipu-glm-tools 上游繁忙失败,最终开自动维修票。

根因:当时 `frontend_designer` 的声明式模型路由只有 GLM 单一候选;zhipu-glm 执行节点升级到 zhipu-glm-tools 后,中文 provider 忙文案未被充分分类为 `http_429/model_unavailable`,cli-runner 没有第二候选可用,ceo-worker 把它当普通 node_failed 快速重试并耗尽 `NODE_FAILURE_MAX_RETRY`。

避免方法:GLM/tool-harness 角色保留低成本首选时,必须配置可执行的跨 runner fallback;中文 provider 忙/限流词必须进入 failover 分类,用同型回归锁住 `zhipu-glm -> zhipu-glm-tools -> 中文繁忙 -> codex`。维修已给 `frontend_designer` 补 codex 兜底并扩展 failover 分类。残余问题不是单票修复,而是全角色 prefer 链审计与低优先级任务策略。

### P-3 · 自愈事件减少终态失败,但不能替代趋势指标

窗口内许多 runner.failover 没有变成 `task.failed`,说明 failover 在发挥作用;但 ui_optimizer 仍出现多次成对 runner_error failover,repair-lead 仍有 timeout failover,insight-scout 也有 runner_error。一次 `watchdog.restart.done` 说明看门狗仍在介入 repair 心跳。

根因:当前复盘若只看 `task.failed` 会漏掉自愈成本和临界信号。failover 需要按供应商/角色/小时聚合,watchdog restart 需要与 daily 点火、repair 长任务、心跳来源关联;否则系统会在没有 terminal failure 的情况下持续消耗兜底 runner 或反复重启。

避免方法:继承 NR14 的可观测性规则,日复盘必须保留 failover top agent/reason 与 watchdog restart 软信号;连续复发时转给主管/维修员做路由或心跳根因,governance 不直接修。

### P-4 · 技能/模块入口缺失是未来复盘可靠性缺口

本次按要求使用 `module-registry` 技能时,技能正文指向的 root registry 和 lookup 脚本均不存在。由于任务输入已经给出明确路径,本轮能降级完成;但未来遇到 Hermes/Yutu/控制台模块化任务时,同一技能会把 agent 引向不存在的入口。

根因:新机迁移后 `~/.codex/modules` 未恢复,但技能仍把它作为强依赖。能力入口声明与本机实际状态不一致,属于「声明未被运行时遵守」的工具链变体。

避免方法:恢复模块注册表或修订技能入口前,复盘类任务要记录“模块入口缺失导致降级”的事实,不能假装已按模块注册表完成路由。此项只列未闭环,不在本复盘中修复。
