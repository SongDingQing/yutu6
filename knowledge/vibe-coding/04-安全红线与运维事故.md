# 04 安全红线 + 运维与事故决策
安全节:**全员**;运维/事故节:repair-lead / it-engineer / orchestrator

## 安全:AI 写的代码默认带漏洞
Veracode 实测 100+ 模型:**45% 的 AI 生成代码含已知漏洞,且不随模型变强而改善**。安全必须由外部流程兜底,不能指望模型自觉,也不能指望换更强的模型。

### 威胁排序(对小型公网 Node 站)
1. **失效的访问控制**:最致命的往往是"某路由忘挂鉴权"——AI 生成 CRUD 常只给 UI 加限制,服务端裸奔
2. **配置错误**:默认配置/调试端口/缺安全头
3. **供应链**:slopsquatting(AI 稳定幻觉的包名被抢注,空包 3 个月 3 万下载)
4. 注入类 RCE:exec 拼接用户输入、路径穿越
5. **密钥泄露**:AI 辅助提交的密钥暴露率 3.2%(人工 1.5%);泄露凭据 70% 三年后仍有效

### 铁律
- 暴露面:防火墙默认拒;内部服务绑 127.0.0.1;误开端口"分钟级"被 Shodan 扫到;部署后外部 nmap 对账,意外新端口=P0
- 密钥:代码零硬编码;**进过 git 历史=已泄露,立即轮换**(删文件救不回);定期全库扫描(玉兔6 对应:security/ 五层机制)
- 命令注入:`exec(cmd+input)` 一律改 `execFile(bin,[args])`;路径穿越:resolve 后校验 startsWith 允许根
- SSRF:正则/黑名单过滤被反复证伪;必须解析后白名单+校验 IP 不落内网段
- **致命三角**:私密数据+不可信内容+对外通信三者齐备=现成外泄通道,至少砍一条;**防御靠架构不靠"教 AI 别上当"**(防护提示词拦 95% 在安全上=不及格)
- 密码存储:绝不明文,快哈希(SHA-256)不合格,首选 Argon2id
- fail-open 专项:grep 所有 catch,确认鉴权/校验失败路径是**拒绝**而不是放过
- 安全必须写进提示词:不显式给安全指令时,模型近半概率选不安全写法

## 运维观测
- **健康检查不骗人**:浅探活(进程在)与深依赖检查(账号库可读写 canary/磁盘余量/下游可达/版本号)分层;"healthz 报 ok 但全站挂 27 分钟"的教训=只探了进程
- **告警最小集五条**:进程崩溃循环 / 磁盘>80%(单机最常见静默故障根因) / 深健康任一项连续失败 / 5xx 比率突增 / 证书剩余<14天。每条必须"可行动",误报堆积三周就被心理屏蔽
- 日志:结构化 JSON 一行一条;错误路径必须留痕;**日志里禁止密钥密码**;journald 设上限
- **备份 3-2-1**:整盘快照≠应用容灾(防不了误删后知后觉/勒索/账号被黑);restic 异地+每月抽样恢复+季度全量演练——**从未恢复过的备份等于没有备份**;备份密码另存

## 事故决策
- **铁律:止血 → 固定证据 → 根因 → 修复 → 复盘**。用户不关心根因只关心恢复;证据留住根因可以慢慢查
- 回滚 vs 前向:刚部署完出问题→回滚;回滚丢数据/格式已迁→前向;根因不明→回滚(不明白的东西打补丁=往黑箱扔变量);老功能坏与部署无关→前向+止损开关
- **区分"没修好"和"余波"**:修复上线后又报障,可能是用户浏览器停在旧页面拿过期令牌撞新代码;止血通报固定加"请刷新再试"
- **免责复盘**:目标是修系统不是找罪人——追责会让人隐瞒信息,失去发现问题的能力。飞轮:事故→清单加条→固化成门或 skill→同类绝迹(玉兔6 对应:experience.md 教训注入)
- 事故中最贵的不是宕机时间,是**错误的确定感**——把每个"应该修好了"换成一次真实路径验证
