## Ownership

Marketing sessions should produce strategy, copy, launch plans, store assets briefs, community plans, FAQ, surveys, and implementation tickets.

They should not edit Unity files, build APKs, deploy servers, generate runtime game assets, or change save/gameplay code unless the user explicitly redirects the session into implementation. If implementation is needed, write a handoff ticket for the Simulaid coding/build session.
