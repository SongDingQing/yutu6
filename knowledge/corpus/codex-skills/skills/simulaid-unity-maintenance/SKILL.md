## Generated Asset Rendering Failures

When a generated image exists on disk but does not render in the Game view, do not assume the PNG itself is bad. Check these in order:

1. Confirm the filename matches the runtime resource name exactly. Card long art uses `Assets/Resources/GeneratedPixel/card_art_{cardId}.png`; code references it extension-free as `card_art_{cardId}`.
2. Confirm the PNG is valid and has the intended dimensions with `file` or another lightweight image check. For long action-card art, expected ratio is 3:1.
3. Inspect import status, not only the raw `.meta` text. Tuanjie/Unity may ignore an asset with an invalid GUID and show Console messages like "does not have a valid GUID and its corresponding Asset file will be ignored." However this user's Tuanjie editor can enable GUID encryption (`com.unity.editor.guid.encryption`), which makes on-disk `.meta` `guid:` values look base64-like even while the import log shows a healthy `using Guid(<32 hex>)`. Do not mass-rewrite encrypted GUID-looking values based only on a regex. Only repair when the Tuanjie log reports an actual invalid-GUID import error, the asset is ignored, or `Resources.Load` still fails after name/cache checks. Do not generate arbitrary GUID strings; let Unity/Tuanjie reimport the asset when possible, or use a valid Unity/Tuanjie-compatible `.meta`.
4. If the UI displays the raw asset name, such as `card_art_awakening`, it means `SimulaidPixelAssets.GetSprite()` returned null and `AddPixelSpriteToFrame()` fell back to text. That usually points to a Resources import/cache issue or a mismatched resource name, not a text/UI layout bug.
5. Remember `SimulaidPixelAssets` caches both loaded sprites and missing sprite names. In the editor, assets copied during Play mode may not be visible to `Resources.Load` until import refresh; clear the cache or rely on the editor-only loose PNG fallback before judging the render.
6. Card art import size matters for clarity. `card_art_*` assets should be imported at a high enough max texture size, currently 2048, so 3:1 card images are not downscaled to blurry thumbnails.
