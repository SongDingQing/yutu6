# knowledge/corpus/ · 原始冷档(不直接被嵌入)

> 记忆三层(蓝图 §11 Hermes 热/温/冷):
> - **冷(corpus/)**:原始迁移记录 / 模块快照 / 历史长文的**durable 归档**。不被 ingest 直接扫。
> - **暖(../../wiki/)**:从冷档**提炼**出的人可读笔记 —— `ingest.py` 只扫 `wiki/`,这层才进向量+图谱。
> - **热**:当前对话上下文。

## 为什么要这层

迁移记录原本在「玉兔搬家部署包」(搬家纸箱,迟早归档/删)。把其中**长期知识**拷进 `corpus/`,
让它脱离纸箱、随工作区 + git + 冷备一起走。这就是"旧积累进新架构槽位"。

## 内容(由 B4 填充)

| 来源 | 落点 |
|---|---|
| 迁移记录 00–08 | `corpus/迁移记录/` |
| module-snapshot(2 模块全文) | `corpus/module-snapshot/` |
| 设计借鉴 / 更新文档 | `corpus/docs/` |

## 怎么进知识库

冷档不自动嵌入。要让某段知识可检索 → 在 `../../wiki/` 写一篇**提炼笔记**(指回 corpus 出处),
再跑 `../build.sh`。提炼的好处:embedding 质量高、图谱抽取准、避免把流水账全灌进向量。
