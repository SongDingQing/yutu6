# 自省优化(Codex 版)

对指定模块做证据化的挑刺-优化闭环。与 codex 侧 `~/.codex/skills/self-reflection-optimizer` 是同一机制的两个执行体:**共用同一套账本格式、分级判据、产物目录和案例沉淀**,不要另造格式。

铁律(来自老板 2026-06-29 对优化师的整改):**禁止"没问题/无需改动"式回复**。每轮至少挑出 3 条带证据的具体问题;真挑不出 3 条,就写明你核查过哪些维度、各维度的证据为什么过关——把"没问题"证明出来,而不是声明出来。

## 0. 预算(老板可随口覆盖)

单文件模块默认:账本 ≤10 条、auto_execute ≤3 条;跨文件/整个子系统:账本 ≤16 条、auto_execute ≤5 条。老板说"验证运行/先别提交"时跳过 git 提交步骤,改动留工作树并在汇报注明。

## 1. 定位模块

老板给了明确路径就直接用;否则先查 `shared/capability_registry/registry.json` 的模块 id/关键词,再工作区检索(rg/Glob)。还有歧义就列出候选让老板选,不要按猜的挑刺。

## 2. 装载背景

挑刺前先读(存在才读):目标模块的 README/module.json/相邻测试;`memory/experience.md` 中与该模块相关的教训(用 grep 定位,别全读);`board/learning-cases/` 相关案例。历史教训是挑刺的放大器——同族问题优先查。
红线:密钥不读不回显;Starlaid/星桥 排除。

## 3. 穷尽挑刺 → 账本

多视角挑:正确性、稳定性(竞态/边界/失败可观测性)、性能(热点/冗余 IO/token 消耗)、紧凑度(重复代码/多处维护的同一常量/死配置)、与既有决策的一致性(`memory/decisions.md`)、测试覆盖、假完成风险(能否不留证据就宣称成功)。
模块大(>2000 行或跨多文件)时,用 Workflow/子代理按视角并行挑,再合并去重。

每条问题写一个账本项(与 codex 版同构):

```markdown
### <问题短标题>
- 证据: <文件:行号 / 日志行 / 测试输出 / 截图路径>
- 影响: <坏在哪:错误/成本/风险/阻碍演进>
- 修法: <具体改动,不要空泛建议>
- 风险: <low|medium|high> + 原因
- 分级: <auto_execute | owner_decision | defer>
- 验证: <可运行的命令/测试/可核对的产物>
```

## 4. 分级判据

**auto_execute**(全部满足才算):收益明确有证据;改动窄、可回滚、局限在该模块;不改公共 API、队列语义、鉴权、路由、持久化格式、现有用户可见行为;有可本地运行的验证;不碰密钥/Starlaid/删除类操作/外部账号。
**owner_decision**(任一满足):收益有争议或取决于产品方向;可能影响现有功能/UI 预期/成本结构;需要迁移、删除、凭据、登录、外部服务配置或高成本调用;验证路径不清。
**defer**:证据不足、超出目标模块、或触碰排除项。
**拿不准一律降级到 owner_decision**——判级宁保守,执行不返工。

## 5. 执行与升级

auto_execute 项逐条做,一条一验:
1. 最小改动;
2. 跑账本里写的验证 + 模块相关测试;改动文件被 `shared/engine/` 或 `server.js`/`ceo-worker.js`/`engine-runner.js` 直接 require 时,跑 `node tests/run.js` 全量(约 1 分钟,值得)。失败立即回滚该条,改判 defer 并记原因;
3. 产物账本写 `projects/控制台/artifacts/self-reflection-optimizer/<模块名>-self-review-<YYYYMMDD>.md`(与 codex 版同一目录);
4. 可复用的教训追加到 `board/learning-cases/self-reflection-optimizer-cases.md`,条目字段照该文件既有格式:来源/任务链/场景/现象/根因/改法/验证/可复用原则;
5. 提交遵守工作区规矩:只 add 声明文件、绝不 `git add -A`、commit 前扫密钥。

owner_decision 项**绝不动手**,汇总成决策清单(见第 6 节格式),等老板拍板。老板回复"同意 2、3"或"全部同意"后,把对应项按 auto_execute 流程执行并补验证。

## 6. 汇报格式

按此顺序输出(极简中文):
1. 目标模块 + 读过的证据源;
2. 挑刺账本(全部问题,含 defer);
3. **已自动执行**:每条附验证输出/证据路径;
4. **待老板拍板**:编号列表,每条一行利、一行弊、一行建议(便于老板回"同意 N");
5. 沉淀的案例与产物路径。

## 硬停止

改鉴权/登录/外部服务配置/凭据位置、删除不可逆数据、影响在跑任务的操作:先停下问老板,任何分级都不豁免。
